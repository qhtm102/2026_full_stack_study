# PRD: 게시판 글쓰기 화면 (Bulletin Board Writing Screen)

## 1. 개요 (Overview)
본 문서는 HTML Form 태그를 활용한 게시판 글쓰기 화면의 기능 및 디자인 요구사항을 정의합니다. 사용자가 게시글을 작성하고 전송할 수 있는 직관적이고 표준화된 인터페이스를 제공하는 것을 목표로 합니다.

## 2. 목표 (Goals)
- 표준 HTML Form 태그를 활용하여 데이터 입력 구조 설계.
- 테이블 레이아웃을 이용한 깔끔한 UI 구성.
- 필수 입력 항목 및 입력 형식을 명확히 안내하여 사용자 편의성 증대.

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 기본 구조
- **태그**: `<form>` 태그를 사용하여 입력 데이터를 그룹화합니다.
- **전송 방식 (Method)**: `post`
- **전송 대상 (Action)**: (없음)

### 3.2 입력 항목 상세
| 항목 | 입력 방식 | 필수 여부 | 상세 조건 |
| :--- | :--- | :---: | :--- |
| 작성자 | `text` | 필수 (*) | placeholder 제공 |
| 비밀번호 | `password` | 필수 (*) | 입력 시 마스킹 처리 |
| 제목 | `text` | 필수 (*) | placeholder 제공 |
| 카테고리 | `select` | - | 옵션: 공지사항, 질문, 자유게시판 |
| 공개 여부 | `radio` | - | 옵션: 공개 / 비공개 |
| 내용 | `textarea` | 필수 (*) | 최소 5줄(rows) 이상 입력 가능하도록 설정 |
| 첨부파일 | `file` | - | 단일 파일 선택 가능 |
| 작성 동의 | `checkbox` | - | 게시글 등록 동의 여부 체크 |

### 3.3 버튼 기능
- **등록 버튼**: 입력된 데이터를 서버로 전송 (Submit).
- **취소 버튼**: 모든 입력 내용을 초기 상태로 리셋 (Reset).

## 4. 비기능 요구사항 & UI/UX (Non-Functional & UI/UX)

### 4.1 접근성 및 레이아웃
- **Labeling**: 모든 입력 항목에 대해 `<label>` 태그를 사용하여 접근성을 확보합니다.
- **필수 표시**: 필수 입력 항목(작성자, 비밀번호, 제목, 내용) 옆에 `*` 기호를 표시합니다.
- **정렬**: `<table>` 태그를 사용하여 항목명과 입력창을 수평으로 정렬하여 가독성을 높입니다.
- **안내**: `placeholder` 속성을 사용하여 사용자에게 입력 힌트를 제공합니다.

### 4.2 화면 구성 예시
1. 상단 중앙에 "게시판 글쓰기" 제목 배치.
2. 테이블 형식을 이용해 [항목명 | 입력 필드] 순으로 배치.
3. 하단 중앙에 [등록] [취소] 버튼 나란히 배치.
